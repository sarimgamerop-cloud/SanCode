## About San

San is a dynamically and strongly typed interpreted programming language that enforces immutability and prevents accidental reassignments.

San is an extremely simple language with Python as its backend. It has the potential to become the best programming language for beginners who are just starting to learn programming. Its dynamic, strong typing helps students learn what they should and should not do.

It also uses conventions from multiple languages, so if you learn San, you learn the following programming language conventions simultaneously:

- Python
- Java
- C
- C++
- Rust
- JavaScript

## Features

San comes with a set of useful features for beginners:

- **Data Types:** San currently supports four data types: `integers`, `floats`, `booleans`, and `strings`.
- **Variables:** You can define mutable variables using `dec` and immutable variables (constants) using `const`.
- **Control Flow:** San has `if` and `else` for conditionals and a `while` loop for iteration.
- **Functions:** San supports functions defined with the `func` keyword.
- **Accidental Reassignment Prevention:** You can only reassign or mutate variables using the `flux` keyword, avoiding unnecessary reassignments.

> San does not support array data types yet; it is still in development.

## Goals

San has clearly defined goals: to be **the best programming language for beginners** who want to get started with programming.

Initially, San started as a high school project by a delusional and broke teenager who did not have money to buy a laptop, so the entire language was made on an Android device.

San also serves as a message to those who cannot afford a laptop for coding: keep pushing. All you need is passion, and the universe will help you with the rest.

## The Lore

### My Story
My initial impressions of computer science were terrible; in fact, up until class 6 it was my weakest subject. Then, in 2021, while the world was still limited due to the COVID-19 pandemic, I spotted the words ***"Programming In Python"*** written in the index of my book, and it startled me, as I wondered what a computer book had to do with "Python" — it is a snake. Then an era began that I never imagined, suddenly I was the best in class at computers. Every line I wrote was somehow correct and compiled with zero errors. That was the development of my character arc.

### The Story of San
First of all, let us discuss why I named it "San." Well, I didn't. The first name I chose was "Arc," which felt cool at first, but after I finished the parser it no longer felt right.

So I chose the word "San," which means "mountain" in Korean, because I was obsessed with mountains.

Initially, San began as just a school project, but then I realised the potential it had to be a great language for beginners. No unnecessary keywords, no difficult syntax decisions, and you become accustomed to many kinds of languages.

**Why a programming language for a school project?**
Now, why did I choose a programming language for a high school project? Nobody cared; I could've made something simpler, like a library management system. That was the issue: everyone else made boring stuff full of AI content. I wanted mine to have an identity. Also, building San for a school project enabled me to learn about languages sooner. If I hadn't, I wouldn't have been able to make time for learning about programming, which I was very interested in.

**Why Python and not a low-level language?**
There are multiple reasons for this:
- First and foremost, the school mandated Python because it was in our curriculum.
- Python is my first language, and I knew it best.
- Python enables fast development due to its simple syntax and support for OOP.
- San was never about speed or performance.

## Target Users

- learning programming
- small prototype programs
- hobbyists who want to learn programming languages
